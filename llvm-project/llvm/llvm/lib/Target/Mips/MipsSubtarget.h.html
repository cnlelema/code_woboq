362 /// This is a very ugly hack. CodeGenPrepare can sink pointer arithmetic to
363 /// appear closer to load and store operations (because SelectionDAG only
364 /// looks at one basic block at a time). Unfortunately, it defaults to using
365 /// ptrtoint / inttoptr pairs, which means that SelectionDAG doesn't generate
366 /// pointer addition nodes and instead just ends up producing CToPtr /
367 /// CFromPtr instruction pairs. There is a command-line flag for preferring
368 /// to sink GEPs instead, but no way for the subtarget to set this flag.
369 /// Claiming that you use AA also enables the GEP-sinking mode, so we claim
370 /// this for CHERI.
371 bool useAA() const override { return IsCheri; }

412 /// Does the system support unaligned memory access.
413 ///
414 /// MIPS32r6/MIPS64r6 require full unaligned access support but does not
415 /// specify which component of the system provides it. Hardware, software, and
416 /// hybrid implementations are all valid.
417 /// CHERI is not MIPSr6, but provides a hybrid implementation where
418 /// in-cache-line unaligned accesses are handled in software and ones
419 /// spanning cache lines are emulated in the OS.
420 bool systemSupportsUnalignedAccess(unsigned AS=0) const {
421 return (IsCheri && AS==200) || hasMips32r6();
422 }
