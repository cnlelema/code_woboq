26 /// ISD::NodeType enum - This enum defines the target-independent operators
27 /// for a SelectionDAG.
28 ///
29 /// Targets may also define target-dependent operator codes for SDNodes. For
30 /// example, on x86, these are the enum values in the X86ISD namespace.
31 /// Targets should aim to use target-independent operators to model their
32 /// instruction sets as much as possible, and only use target-dependent
33 /// operators when they have special requirements.
34 ///
35 /// Finally, during and after selection proper, SNodes may use special
36 /// operator codes that correspond directly with MachineInstr opcodes. These
37 /// are used to represent selected instructions. See the isMachineOpcode()
38 /// and getMachineOpcode() member functions of SDNode.
39 ///

421 /// Shift and rotation operations. After legalization, the type of the
422 /// shift amount is known to be TLI.getShiftAmountTy(). Before legalization
423 /// the shift amount can be any type, but care must be taken to ensure it is
424 /// large enough. TLI.getShiftAmountTy() is i8 on some targets, but before
425 /// legalization, types like i1024 can occur and i8 doesn't have enough bits
426 /// to represent the shift amount.
427 /// When the 1st operand is a vector, the shift amount must be in the same
428 /// type. (TLI.getShiftAmountTy() will return the same type when the input
429 /// type is a vector.)

431 /// amount modulo the element size of the first operand.

553 /// X = FP_ROUND(Y, TRUNC) - Rounding 'Y' from a larger floating point type
554 /// down to the precision of the destination VT. TRUNC is a flag, which is
555 /// always an integer that is zero or one. If TRUNC is 0, this is a
556 /// normal rounding, if it is 1, this FP_ROUND is known to not change the

558 ///
559 /// The TRUNC = 1 case is used in cases where we know that the value will
560 /// not be modified by the node, because Y is not using any of the extra
561 /// precision of source type. This allows certain transformations like

991 /// ISD::CondCode enum - These are ordered carefully to make the bitfields
992 /// below work out, when considering SETFALSE (something that never exists
993 /// dynamically) as 0. "U" -> Unsigned (for integer operands) or Unordered
994 /// (for floating point), "L" -> Less than, "G" -> Greater than, "E" -> Equal
995 /// to. If the "N" column is 1, the result of the comparison is undefined if
996 /// the input is a NAN.
997 ///
998 /// All of these (except for the 'always folded ops') should be handled for
999 /// floating point. For integer, only the SETEQ,SETNE,SETLT,SETLE,SETGT,
1000 /// SETGE,SETULT,SETULE,SETUGT, and SETUGE opcodes are used.
1001 ///
1002 /// Note that these are laid out in a specific order to allow bit-twiddling
1003 /// to transform conditions.
1004 enum CondCode {
1005 // Opcode N U L G E Intuitive operation
1006 SETFALSE, // 0 0 0 0 Always false (always folded)
1007 SETOEQ, // 0 0 0 1 True if ordered and equal
1008 SETOGT, // 0 0 1 0 True if ordered and greater than
1009 SETOGE, // 0 0 1 1 True if ordered and greater than or equal
1010 SETOLT, // 0 1 0 0 True if ordered and less than
1011 SETOLE, // 0 1 0 1 True if ordered and less than or equal
1012 SETONE, // 0 1 1 0 True if ordered and operands are unequal
1013 SETO, // 0 1 1 1 True if ordered (no nans)
1014 SETUO, // 1 0 0 0 True if unordered: isnan(X) | isnan(Y)
1015 SETUEQ, // 1 0 0 1 True if unordered or equal
1016 SETUGT, // 1 0 1 0 True if unordered or greater than
1017 SETUGE, // 1 0 1 1 True if unordered, greater than, or equal
1018 SETULT, // 1 1 0 0 True if unordered or less than
1019 SETULE, // 1 1 0 1 True if unordered, less than, or equal
1020 SETUNE, // 1 1 1 0 True if unordered or not equal
1021 SETTRUE, // 1 1 1 1 Always true (always folded)
1022 // Don't care operations: undefined if the input is a nan.
1023 SETFALSE2, // 1 X 0 0 0 Always false (always folded)
1024 SETEQ, // 1 X 0 0 1 True if equal
1025 SETGT, // 1 X 0 1 0 True if greater than
1026 SETGE, // 1 X 0 1 1 True if greater than or equal
1027 SETLT, // 1 X 1 0 0 True if less than
1028 SETLE, // 1 X 1 0 1 True if less than or equal
1029 SETNE, // 1 X 1 1 0 True if not equal
1030 SETTRUE2, // 1 X 1 1 1 Always true (always folded)
1031
1032 SETCC_INVALID // Marker value.
1033 };
