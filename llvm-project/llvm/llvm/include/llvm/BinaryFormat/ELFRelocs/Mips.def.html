65// XXXAR: We use this to write an absolute value to a given offset. Plain MIPS does
66// this by writing the value to GOT slot and using a R_MIPS_REL32 where the r_symndx points
67// to the GOT entry that should have been relocated to point to the correct value and then load that to write the value.
68// This seems a) unnecessarily complicated and b) not to always work since somehow we were getting
69// those relocations emitted where r_symndx pointed to the local part of the GOT so the relocation
70// would just add the load address. Furthermore for CheriABI we really don't want a GOT but still want to be able
71// to write absolute 64-bit values
72ELF_RELOC(R_MIPS_CHERI_ABSPTR, 70)
73// A dynamic relocation that writes the st_size of a symbol (currently used to
74// support __cap_relocs and .global_sizes against unresolved symbols)

105// RTLD should set the following to a capability pointing to symbol.
106// These relocations should be processed immediatedly at load time.
107// Note: All R_MIPS_CHERI_CAPABILITY relocations for STT_FUNC symbols must
108// resolve to the same address to ensure function pointer comparison works as
109// required by the C standard. If a function is only called but the address is
110// not taken, the static linker should emit a R_MIPS_CHERI_CAPABILITY_CALL
111// relocation instead since this allows RTLD to only perform the symbol lookup
112// on the first call rather than at load time
113ELF_RELOC(R_MIPS_CHERI_CAPABILITY, 90)
114// The same as R_MIPS_CHERI_CAPABILITY but RTLD may use lazy binding.
115ELF_RELOC(R_MIPS_CHERI_CAPABILITY_CALL, 91)

117// Set the tag bit on the referenced data (must point within the current object)
118// Must not be preemtible!
119ELF_RELOC(R_MIPS_CHERI_SETTAG, 92)
120// Set the tag bit after adding the ELF object load address to the raw capability bits
121ELF_RELOC(R_MIPS_CHERI_SETTAG_LOADADDR, 93)
122// Write a capability pointing to the capability table for the referenced symbol
123// and set the tag bit (if processed by rtld). When linking statically we can
124// fill in the bits and change it to a R_MIPS_CHERI_SETTAG
125// When linking dynamically it should always be possible to change this to a
126// R_MIPS_CHERI_SETTAG_LOADADDR relocation
127// The same as R_MIPS_CHERI_CAPABILITY but must not be preemptible
128// The linker should always be able to optimize this into a
129// R_MIPS_CHERI_SETTAG_LOADADDR or R_MIPS_CHERI_SETTAG
130ELF_RELOC(R_MIPS_CHERI_LOCAL_CAPABILITY,94) // XXXAR: probably not needed
