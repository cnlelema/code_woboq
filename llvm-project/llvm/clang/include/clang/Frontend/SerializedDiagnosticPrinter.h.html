28/// a bitcode file.
29///
30/// The created DiagnosticConsumer is designed for quick and lightweight
31/// transfer of diagnostics to the enclosing build system (e.g., an IDE).
32/// This allows wrapper tools for Clang to get diagnostics from Clang
33/// (via libclang) without needing to parse Clang's command line output.
34///
